Encapsulation

  • Encapsulation:

    In object-oriented computer programming (OOP) languages, the notion of encapsulation (or OOP Encapsulation) refers to the binding of data, along with the methods that operate on that data, into a single unit.

    • Examples:

      1. The general idea of this mechanism is simple. For example, you have an attribute that is not visible from the outside of an object. You bundle it with methods that provide read or write access. Encapsulation allows you to hide specific information and control access to the internal state of the object.

      2. If you’re familiar with any object-oriented programming language, you probably know these methods as getter and setter methods. As the names indicate, a getter method retrieves an attribute and a setter method changes it. Depending on the methods that you implement, you can decide if an attribute can be read and changed. You may also control if the attribute is read-only or not visible at all. Later, we’ll show you how the setter method can also be used to implement additional validation rules to ensure that your object always has a valid state.
